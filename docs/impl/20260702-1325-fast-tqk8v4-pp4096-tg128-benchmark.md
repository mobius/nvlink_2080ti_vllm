# fast/tqk8v4 PP4096/TG128 实测

- 时间：2026-07-02 13:25 CST
- 路线：weicj vLLM-2080Ti-Definitive v0.1.11
- 模型：Qwen3.6-27B-AWQ-INT4
- GPU：2×RTX 2080Ti 22G (NVLink)，TP=2
- 编译器：g++-12 12.3.0
- Profile：`qwen27b/fast/int4/tqk8v4-256K-mtp3-text-only.env`
- 模式：fast
- 关键参数：MTP=3, FlashQLA legacy GDN prefill, **TurboQuant KV k8v4**, FULL_AND_PIECEWISE cudagraph, GPU_UTIL=0.90

## 启动命令

```bash
cd /mnt/hdd_storage/vllm_2080ti/weicj-vllm-2080ti
env CUDA_DEVICE_ORDER=PCI_BUS_ID CUDA_VISIBLE_DEVICES=0,1 GPU_DEVICES=0,1 TP_SIZE=2 \
    CC=/usr/bin/gcc-12 CXX=/usr/bin/g++-12 CUDAHOSTCXX=/usr/bin/g++-12 \
    QUANTIZATION=compressed-tensors \
    HF_HOME=/mnt/hdd_storage/vllm_2080ti/cache/hf \
    TRITON_CACHE_DIR=/mnt/hdd_storage/vllm_2080ti/cache/triton_weicj_tqk8v4_v2 \
    MODEL_DIR=/mnt/hdd_storage/vllm_2080ti/models/Qwen3.6-27B-AWQ-INT4 \
    PROFILE=qwen27b/fast/int4/tqk8v4-256K-mtp3-text-only.env \
    MODE=fast PORT=8000 SERVICE_SCOPE=local \
    ./launcher.sh --non-interactive
```

## 实测结果

| 运行 | prompt tokens | gen tokens | TTFT (s) | prefill tok/s | decode tok/s | 备注 |
|------|---------------|------------|----------|---------------|--------------|------|
| run1 | 4096 | 128 | 3.139 | 1304.8 | 89.97 | 含 warm-up |
| run2 | 4096 | 128 | 2.609 | 1569.7 | 91.13 | warm 后 |
| run3 | 4096 | 128 | 2.609 | 1569.9 | 90.59 | warm 后 |

- **稳定 decode 吞吐：约 90.5-91.1 tok/s**
- 峰值单点：91.13 tok/s

## 对比

| 配置 | PP4096/TG128 decode tok/s | 差距 |
|------|---------------------------|------|
| caovan v0.1.3 (256K, MTP=2) | ~40.1 | baseline |
| weicj normal/fp16kv/MTP=3 | ~68 | +70% |
| weicj fast/tqk8v4/MTP=3 | ~90.8 | +126% vs caovan |
| weicj 宣称峰值 | 101.3 | 还差 ~10.5 tok/s |

## 结论

1. TurboQuant KV (k8v4) + fast mode + FULL_AND_PIECEWISE cudagraph 在 PP4096/TG128 口径下把吞吐从 68 tok/s 提升到约 91 tok/s，提升约 33%。
2. 仍未达到 weicj 宣称的 101.3 tok/s。
3. 下一步尝试：提高 `GPU_UTIL` 到 0.94（fast/fp16kv profile 默认值），或对齐 weicj lock file / patches。
